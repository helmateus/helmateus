## Como eu penso trabalho

Alguns princípios que guiam meus projetos:

- software precisa servir à operação, não o contrário
- interface boa reduz erro, atrito e retrabalho
- dado sem contexto não decide nada
- automação boa é a que simplifica o processo real
- IA precisa entrar onde gera ganho concreto

Também gosto de estruturar repositórios com documentação operacional, contexto, backlog e material de decisão para manter a execução organizada ao longo do tempo.
